# CascadeOps Implementation Plan

## Scope rule

Finish one complete, source-linked policy-change compilation before adding breadth. P0 proves the refund-window change across five curated artifacts in Replay and one bounded Live GPT-5.6 run.

## Milestones

### M0 — Pre-production

Rules matrix, blueprint, PRD, user journeys, UX/content specifications, architecture, ADRs, threat model, testing/evaluation plans, demo story and submission plan.

Exit: documents agree on the same workflow, contracts, truthfulness boundary and exclusions.

### M1 — Foundation

Strict TypeScript Next.js app, quality scripts, server-only environment validation, professional responsive shell, Replay/Live provider interface.

Exit: credential-free lint, typecheck, tests and build pass.

### M2 — Contracts and fixtures

Canonical refund policy, five dependent artifacts, strict schemas, deterministic expected impacts/patches, validators and contract tests.

Exit: unknown IDs, fabricated citations, unsafe replacements and invalid state transitions fail closed.

### M3 — Golden Replay compiler

Policy comparison, impact cascade, patch review, approvals, verification and compilation receipt.

Exit: deterministic judge path completes by keyboard on desktop and mobile layouts.

### M4 — Live GPT-5.6

Bounded server-only Responses API provider using Structured Outputs and `store: false`, visible provenance, typed failures and redacted smoke receipt.

Exit: one live compilation validates against the same contracts without exposing secrets.

### M5 — Hardening and evaluation

Contract, adversarial, security, accessibility, browser and repeated replay tests; secret and dependency/license audits.

Exit: all hard grounding and approval graders pass.

### M6 — Public delivery

Public repository, clean-clone quickstart, deployed Replay application and judge-perspective verification.

### M7 — Demo package

Real screenshots, architecture diagram, thumbnail, narrated video under three minutes and captions.

### M8 — Submission

Devpost content, links, Codex session ID, category, eligibility, final submission and independent receipt verification.

## Cut order

Cut decorative animation, extra policy scenarios, arbitrary uploads, connector previews and export formats before cutting schema validation, citations, approval gates, Replay mode, accessibility, verification or submission evidence.
