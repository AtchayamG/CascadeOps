A single approved policy change can leave five operational documents telling five different stories. CascadeOps turns that follow-through into a reviewable compilation workflow.

This public demo uses deterministic simulated fixture data, clearly labelled Replay Mode. Our source refund clause changes from thirty days to fourteen, with stable clause and change identifiers.

CascadeOps traces that clause through a curated set of dependencies and proposes five bounded edits: a support S O P, request form, response template, Q A checklist, and training guide. Every proposal cites its exact target anchor and source change.

Human authority is enforced, not implied. I can reject a patch, and candidate compilation fails closed. I can revise that decision before compilation. Only when all five proposals are explicitly approved does the workflow continue.

Now CascadeOps applies all five replacements atomically to isolated, in-memory candidate copies. The state says candidate compiled, not verified. No enterprise file or repository is modified.

Verification is a separate, model-free action. Twenty deterministic assertions check that stale values are absent, the new value is present, anchors remain intact, and untouched blocks are unchanged. Only then do the candidates become verified.

The receipt records the run, decision totals, and assertion evidence. Its S H A two fifty six value is a content checksum for integrity comparison, not a digital signature or compliance seal.

Codex served as principal engineer: architecture, strict contracts, compiler, interface, worker coordination, tests, and final evidence audit. Optional Live Mode uses G P T five point six Responses Structured Outputs with request storage disabled through store false, strict validation, and no silent fallback. A bounded local live smoke proof passed.
