# CascadeOps Demo Script

- Target duration: 2:30
- Public mode shown: Simulated Replay
- Narration: calm professional operations lead

## Storyboard and voiceover

### 0:00-0:10 - Title

**Visual:** CascadeOps thumbnail/title card, then production app home.

**Voiceover:** "A single approved policy change can leave five operational documents telling five different stories. CascadeOps turns that follow-through into a reviewable compilation workflow."

### 0:10-0:27 - Source change and honest mode label

**Visual:** Point to the persistent Replay banner and the source policy cards showing 30 days changed to 14 days.

**Voiceover:** "This public demo uses deterministic simulated fixture data, clearly labelled Replay Mode. Our source refund clause changes from thirty days to fourteen, with stable clause and change identifiers."

### 0:27-0:47 - Trace impacts

**Visual:** Click **Compile Policy Change**. Show five proposal cards and exact artifact/anchor citations.

**Voiceover:** "CascadeOps traces that clause through a curated set of dependencies and proposes five bounded edits: a support SOP, request form, response template, QA checklist, and training guide. Every proposal cites its exact target anchor and source change."

### 0:47-1:06 - Human gate

**Visual:** Reject the first proposal, click **Compile Approved Candidates** to show the blocked state, then change it to approve and approve the remaining four.

**Voiceover:** "Human authority is enforced, not implied. I can reject a patch, and candidate compilation fails closed. I can revise that decision before compilation. Only when all five proposals are explicitly approved does the workflow continue."

### 1:06-1:22 - Compile isolated candidates

**Visual:** Click **Compile Approved Candidates**. Pause on the APPLIED state and five **Candidate Compiled** badges.

**Voiceover:** "Now CascadeOps applies all five replacements atomically to isolated in-memory candidate copies. Notice the state says candidate compiled, not verified. No enterprise file or repository is modified."

### 1:22-1:41 - Deterministic verification

**Visual:** Click **Run Verification Check**. Show the verified banner and assertion list.

**Voiceover:** "Verification is a separate, model-free action. Twenty deterministic assertions check that stale values are absent, the new value is present, anchors remain intact, and untouched blocks are unchanged. Only then do the candidates become verified."

### 1:41-1:56 - Receipt

**Visual:** Click **Export Compilation Receipt**. Show the two-column summary, evidence list, and wrapped checksum. Optionally click **Download JSON Receipt**.

**Voiceover:** "The receipt records the run, decision totals, and assertion evidence. Its SHA-two-fifty-six value is a content checksum for integrity comparison, not a digital signature or compliance seal."

### 1:56-2:24 - Codex and GPT-5.6

**Visual:** Show a concise architecture slide or repository README and the live-smoke evidence heading.

**Voiceover:** "Codex served as principal engineer: architecture, strict contracts, compiler, interface, worker coordination, tests, and final evidence audit. Optional Live Mode uses GPT-five-point-six Responses Structured Outputs with request storage disabled through store false, strict validation, and no silent fallback. A bounded local live smoke proof passed."

### 2:24-2:30 - Close

**Visual:** Return to thumbnail/title card.

**Voiceover:** "CascadeOps: one policy change, every operation aligned."
